Create variants by passing an object to the variants key like in [cva](https://cva.style/getting-started/variants/). The key should match the prop name and the value should be a function that returns a string. You could also re-use the props in the function.

## Example

```ts choice="react"
import ma from "@marmo/react"

interface AlertProps {
  $severity: "info" | "warning" | "error"
  $isActive?: boolean
}

export const Alert = cm.div.variants<AlertProps>({
  // optional base
  base: ({ $isActive }) => `
    ${isActive ? "custom-active" : "custom-inactive"}
    p-4 rounded-md
  `,
  // required
  variants: {
    $severity: {
      warning: "bg-yellow-100 text-yellow-800",
      info: ({ $isActive }) => `bg-blue-100 text-blue-800 ${$isActive && "shadow-lg"}`,
      error: ({ $isActive }) => `bg-red-100 text-red-800 ${$isActive && "ring ring-red-500"}`,
    },
  },
  // optional - used if no variant was found
  defaultVariant: {
    $severity: "info",
  },
})
```

### With Wrapper

```jsx
import { Alert } from "./components/Alert"

const MyComponent = () => {
  return (
    <Alert $severity="error" $isActive>
      This is an error alert
    </Alert>
  )
}
```

#### Renders to

```html title="output.html"
<div class="custom-active p-4 rounded-md bg-red-100 text-red-800 ring ring-red-500">
  This is an error alert
</div>
```

### Handles

`variants` - required
: Can define multiple variants for the component. The key should match the prop name and the value should be a function that returns a string. 
: Properties are available too

`base` - optional
: let's you define a base class for the component.
: you can use the props in the base function to conditionally apply classes.

`defaultVariant` - optional
: this is optional. You can define a default variant for the component. If no variant is found, the default variant will be used. The key should match the prop name and the value should
